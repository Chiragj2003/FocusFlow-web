Version: 1.2
Purpose: detailed, task-based prompts for building FocusFlow with Supabase and Clerk.

Core API - Route Handlers
I am building a habit tracking app named FocusFlow using Next.js 15 App Router.

Global API rules
1. Write code only inside src/app/api/**.
2. Use zod for input validation in every route.
3. Use Clerk auth() and scope all queries by userId.
4. Return JSON with { data, error }.
5. Use standard PrismaClient with Supabase DATABASE_URL.
6. Use date-fns and date-fns-tz for timezone normalization.
7. Use 401 for unauthenticated, 400 for invalid input, 404 for missing data.
8. Always validate required fields and coerce types where needed.

Habits API
Your task is to implement the API for this module.
Files to edit:
- src/app/api/habits/route.ts
- src/app/api/habits/[id]/route.ts
- src/app/api/habits/ai-generate/route.ts
Steps:
- Create GET /api/habits with optional includeArchived and category filters.
- Create POST /api/habits to add a habit.
- Create GET /api/habits/[id] to fetch a single habit.
- Create PATCH /api/habits/[id] to update habit fields.
- Create DELETE /api/habits/[id] to archive or delete a habit.
- Create POST /api/habits/ai-generate to generate a habit from a prompt.
Validation:
- goalType must be binary, duration, or quantity.
- if goalType is duration or quantity, require goalTarget and unit.
- title is required and trimmed.
- color must be a valid hex string.
Behavior:
- Always scope by userId.
- Use soft delete for archive unless hardDelete is true.
- AI fallback uses deterministic template matching if GEMINI_API_KEY is missing.
